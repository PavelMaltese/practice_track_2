# Генератор синтетических данных студентов

---

# Что генерируется

Каждая строка набора данных соответствует:

> одному студенту, одному предмету и одному семестру

Набор данных включает:

| Поле | Описание |
|---|---|
| `student_id` | ID студента |
| `gender` | Пол студента (`M` / `F`) |
| `course` | Курс обучения (1–4) |
| `age` | Возраст |
| `semester` | Семестр |
| `subject` | Название предмета |
| `exam_grade` | Оценка за основной экзамен |
| `retake_grade` | Оценка за пересдачу |
| `commission_grade` | Оценка комиссии |
| `attendance_percent` | Процент посещаемости |
| `assignments_on_time_percent` | Процент работ, сданных в срок |
| `retakes_count_semester` | Количество пересдач за семестр |
| `expelled` | Флаг отчисления (`1` — студент отчислен по данному предмету) |
| `student_type` | Тип студента |

---

# Типы студентов

Генератор моделирует 4 категории студентов.

## Распределение типов студентов

| Тип | Вероятность |
|---|---|
| `top` | 5% |
| `good` | 15% |
| `average` | 55% |
| `weak` | 25% |

---

## Характеристики категорий

### `top` — отличники
Особенности:
- практически всегда получают `A`
- высокая посещаемость
- почти всегда сдают работы вовремя
- практически не имеют пересдач

### `good` — хорошие студенты
Особенности:
- чаще получают `A` и `B`
- иногда получают `C`
- редко попадают на пересдачи
- высокая дисциплина

### `average` — средние студенты
Особенности:
- основной диапазон оценок: `B`, `C`, `D`
- иногда получают `A`
- возможны пересдачи
- средняя посещаемость

### `weak` — слабые студенты
Особенности:
- часто получают `D`, `E`, `F`
- высокая вероятность пересдач
- повышенный риск отчисления
- низкая посещаемость

Оценки генерируются случайным образом с использованием вероятностных распределений.

|---|---|
| A | 100% |

# Посещаемость и сдача работ

Для генерации посещаемости и процента работ, сданных в срок, используется нормальное распределение:

```python
np.random.normal(mean, std)
```

где:
- `mean` — среднее значение
- `std` — стандартное отклонение

Это позволяет моделировать естественный разброс поведения студентов.

# Гендерное распределение

Используются следующие пропорции:
- мужчины — 67%
- женщины — 33%

Для студенток дополнительно применяется коэффициент:
- посещаемость × 1.2
- процент сдачи работ в срок × 1.2

Это создаёт статистически заметные различия между группами студентов.

# Логика учебного процесса

В проекте реализована упрощённая модель учебного процесса:

- 4 курса обучения
- 2 семестра на каждом курсе
- 4 экзамена в семестр:
  - Math
  - Physics
  - Programming
  - English

---

## Механика пересдач

Если студент получает `F`:
1. назначается пересдача
2. при повторном `F` назначается комиссия
3. при `F` на комиссии студент отчисляется

После отчисления:
- текущий семестр сохраняется полностью
- последующие семестры не генерируются

# Реалистичность данных

Для повышения реалистичности:
- значения посещаемости различаются по предметам
- используется случайный шум
- возраст увеличивается по мере перехода между курсами
- присутствует небольшая вероятность студентов старшего возраста
