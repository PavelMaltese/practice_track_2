# Генератор синтетических данных студентов

---

# Что генерируется

Каждая строка набора данных соответствует:

> одному студенту, одному предмету и одному семестру

Набор данных включает:

| Поле | Описание |
|---|---|
| `student_id` | ID студента |
| `gender` | Пол студента (`M` / `F`) |
| `course` | Курс обучения (1–4) |
| `age` | Возраст |
| `semester` | Семестр |
| `subject` | Название предмета |
| `exam_grade` | Оценка за основной экзамен |
| `retake_grade` | Оценка за пересдачу |
| `commission_grade` | Оценка комиссии |
| `attendance_percent` | Процент посещаемости |
| `assignments_on_time_percent` | Процент работ, сданных в срок |
| `retakes_count_semester` | Количество пересдач за семестр |
| `expelled` | Флаг отчисления (`1` — студент отчислен по данному предмету) |
| `student_type` | Тип студента |

---

# Типы студентов

Генератор моделирует 4 категории студентов.

## Распределение типов студентов

| Тип | Вероятность |
|---|---|
| `top` | 5% |
| `good` | 15% |
| `average` | 55% |
| `weak` | 25% |

---

## Характеристики категорий

### `top` — отличники
Особенности:
- практически всегда получают `A`
- высокая посещаемость
- почти всегда сдают работы вовремя
- практически не имеют пересдач

### `good` — хорошие студенты
Особенности:
- чаще получают `A` и `B`
- иногда получают `C`
- редко попадают на пересдачи
- высокая дисциплина

### `average` — средние студенты
Особенности:
- основной диапазон оценок: `B`, `C`, `D`
- иногда получают `A`
- возможны пересдачи
- средняя посещаемость

### `weak` — слабые студенты
Особенности:
- часто получают `D`, `E`, `F`
- высокая вероятность пересдач
- повышенный риск отчисления
- низкая посещаемость

# Посещаемость и сдача работ

Для генерации посещаемости и процента работ, сданных в срок, используется нормальное распределение:

```python
np.random.normal(mean, std)
```

где:
- `mean` — среднее значение
- `std` — стандартное отклонение

Это позволяет моделировать естественный разброс поведения студентов.

# Гендерное распределение

Используются следующие пропорции:
- мужчины — 67%
- женщины — 33%

Для студенток дополнительно применяется коэффициент:
- посещаемость × 1.2
- процент сдачи работ в срок × 1.2

Это создаёт статистически заметные различия между группами студентов.

# Логика учебного процесса

В проекте реализована упрощённая модель учебного процесса:

- 4 курса обучения
- 2 семестра на каждом курсе
- 4 экзамена в семестр:
  - Math
  - Physics
  - Programming
  - English

---

## Механика пересдач

Если студент получает `F`:
1. назначается пересдача
2. при повторном `F` назначается комиссия
3. при `F` на комиссии студент отчисляется

После отчисления:
- текущий семестр сохраняется полностью
- последующие семестры не генерируются

## 📊 Метрики и индексы

### 1. Индекс риска студента (взвешенный)

Индекс риска рассчитывается как сумма взвешенных признаков академического неблагополучия:

| Признак | Условие | Вес |
|---------|---------|-----|
| Низкая посещаемость | `attendance_percentage < 70%` | **1** |
| Систематические просрочки | `assignments_on_time_percent < 70%` | **1** |
| Наличие пересдач | `retakes_count_semester > 0` | **2** |
| Плохая оценка | `exam_grade IN {"F","E","D"}` | **3** |

## 2. Индекс риска по предмету

Аналогичный индекс, но вычисляется в контексте конкретного предмета для каждого студента. Используется в тепловой карте.

## 3. Числовое преобразование оценок

Для расчёта среднего балла и построения трендов используется следующая шкала соответствия:

| Буквенная оценка | Числовое значение |
|-----------------|------------------|
| A               | 5                |
| B               | 4                |
| C               | 3                |
| D               | 2                |
| E               | 1                |
| F               | 0                |

## 4. Средний балл за семестр

## 📈 Ключевые показатели (KPI)

| Метрика | Формула | Описание |
|---------|---------|----------|
| Total Students | `DISTINCTCOUNT('Sheet1'[student_id])` | Общее количество студентов |
| Students at Risk | `SUMX(VALUES('Sheet1'[student_id]), IF([Risk Index Weighted] >= 3, 1, 0))` | Количество студентов в зоне риска |
| Avg Risk Index | `AVERAGEX(VALUES('Sheet1'[student_id]), [Risk Index Weighted])` | Средний индекс риска по всем студентам |
| Risk % | `DIVIDE([Students at Risk], [Total Students], 0)` | Доля студентов в зоне риска |

## 🎨 Шкала риска

| Диапазон | Уровень | Цвет | Описание | Рекомендации |
|----------|---------|------|----------|-------------|
| 0 | Стабилен | 🟢 | Отличная успеваемость, нет проблем | Мониторинг, без вмешательства |
| 1–2 | Зона внимания | 🟡 | Есть незначительные проблемы | Рекомендательная беседа |
| 3–4 | Высокий риск | 🟠 | Серьёзные проблемы с успеваемостью | Обязательная встреча с куратором |
| 5+ | Критический риск | 🔴 | Высокая вероятность отчисления | Немедленное вмешательство |

## 📈 Тренд успеваемости
